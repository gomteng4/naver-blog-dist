시작하기
⚙️ 초기 설정
처음 사용자를 위한 단계별 설정 안내입니다.
  1. 1
    zip 다운로드 → 압축 해제 상단 다운로드 버튼으로 zip 파일을 받아 바탕화면(또는 원하는 위치)에 압축 해제합니다.
    naver-blog/ 폴더가 생성됩니다.
  2. 2
    naver-blog.exe 실행 폴더 안의 naver-blog.exe를 실행합니다.
    Windows Defender 경고가 뜨면 추가 정보 → 실행을 클릭하세요.
  3. 3
    라이선스 키 입력 프로그램 실행 시 라이선스 키 입력창이 표시됩니다. 발급받은 키를 입력하고 확인합니다.
  4. 4
    계정 추가 [계정 추가] 버튼 → 네이버 아이디 / 비밀번호 입력.
    블로그 ID가 네이버 아이디와 다를 경우 블로그 ID도 함께 입력합니다.
    예: blog.naver.com/myid123 → 블로그 ID는 myid123
  5. 5
    Gemini API 키 입력 (AI 생성 사용 시) 상단 [설정] 탭 → Gemini API 키를 입력합니다.
    Google AI Studio에서 무료로 발급받을 수 있습니다.
  6. 6
    포스팅 실행 계정 선택 → 날짜 폴더 선택 → [포스팅 시작].
    Chrome이 자동으로 열리고, 네이버에 로그인 후 포스팅을 진행합니다.
    첫 실행 시 Chrome 드라이버가 자동 설치됩니다.
주요 기능
🤖 AI 글 생성
제목 목록과 작성 지침을 준비해두면 Gemini AI가 제목을 한 개씩 꺼내
블로그 글(MD)과 이미지 프롬프트를 동시에 자동 생성합니다.
준비 파일
📄 guide.md — 필수
브랜드 정보, 요금제, 타겟 독자, 금지사항 등을 작성합니다.
계정 추가 시 빈 파일이 자동 생성됩니다.
📋 titles.txt — 필수
제목을 한 줄에 하나씩 작성합니다. 프로그램이 순서대로 꺼내 사용합니다.
가경동 선불폰 개통 어떻게 하나요?
북가좌동 선불폰 KT망 vs LGU+망
명동 선불유심 약정 없이 쓰기 좋은 이유
… (한 줄에 제목 하나씩)
실행 흐름
  1. 1
    미사용 제목 1개 선택 titles.txt에서 순서대로(또는 무작위로) 꺼냅니다. 모두 사용하면 알림이 표시됩니다.
  2. 2
    Gemini가 블로그 글 작성 제목 + guide.md + 선택된 스타일 탭의 섹션 템플릿 → Gemini API → 블로그 본문(MD) 생성.
  3. 3
    이미지 프롬프트 자동 작성 본문의 [이미지 1]~[이미지 4] 배치를 분석해 각 이미지 프롬프트 + 썸네일 프롬프트를 자동 작성합니다.
  4. 4
    날짜 폴더 자동 생성 YYYYMMDD/ 폴더가 생성되고 {제목}.md + image_prompts.txt가 저장됩니다.
    같은 날 여러 개 생성 시 YYYYMMDD_01/, YYYYMMDD_02/ 형식으로 번호가 붙습니다.
  5. 5
    이미지 생성 → 포스팅 실행 날짜 폴더를 선택하고 [포스팅 시작]을 누르면 이미지 생성 → 블로그 업로드까지 한 번에 진행됩니다.
guide.md 작성 팁 — 자료 다운로드에서 guide_만들기_프롬프트.md를 받아
ChatGPT·Claude·Gemini에 붙여넣고 서비스 정보를 채우면 guide.md가 자동 완성됩니다.
💡 "표로 정리해줘" 같은 포맷 지시는 넣지 마세요 — styles/ 폴더가 담당합니다.

주요 기능 · NEW v1.4.25
✨ 스타일 탭 (프로필)
블로그 주제마다 독립된 스타일 프로필을 만들고,
각 탭의 섹션 템플릿(.md)을 랜덤 선택해 글 스타일을 다양하게 유지합니다.
스타일 탭이란?
탭 예시
기본 / 부동산 / 여행 / 음식
탭마다 독립적인 섹션 구성 + 스타일 파일을 가집니다.
탭 관리
앱 [스타일] 탭에서 추가 / 이름 변경 / 삭제.
탭 전환 시 섹션 목록이 해당 탭 설정으로 바뀝니다.
폴더 구조
prompt/styles/
  ├── 기본/                    기본 스타일 탭
  │   ├── 도입부/              공감형.md, 질문형.md …
  │   ├── 요금제/              표형식.md, 리스트형식.md
  │   ├── 개념_장점/           장점나열형.md …
  │   └── 마무리/              요약형.md, 행동유도형.md
  └── 부동산/                  추가 스타일 탭 (예시)
        └── 도입부/              부동산용 도입부 템플릿
guide.md vs styles/ 역할 분리
📄 guide.md — "무엇을"
  • 브랜드명, URL, 서비스명
  • 요금제 정보, 가격, 혜택
  • 타겟 독자, 금지사항
📁 styles/ — "어떻게"
  • 도입부 스타일 (공감/질문/상황)
  • 요금제 표시 형식 (표/리스트)
  • 전체 구조, 문체, 섹션 포맷
⚠️ guide.md에 포맷 지시를 넣지 마세요.
"표로 정리해줘", "Q&A 형식으로 써줘" 같은 내용은 styles/ 폴더의 .md 파일에 작성하세요.
guide.md와 styles/가 충돌하면 글 품질이 저하됩니다.
스타일 파일 관리 방법
  1. 앱 [스타일] 탭 → 스타일 프로필(탭) 추가/전환
  2. 섹션 목록에서 📂 폴더 버튼 클릭 → 탐색기로 해당 폴더 열기
  3. 폴더 안의 .md 파일을 메모장 등으로 수정하거나 새 .md 파일 추가
  4. [🌐 커뮤니티 템플릿] 버튼으로 다른 사용자 템플릿을 가져올 수도 있습니다
주요 기능 · NEW v1.4.25
🌐 커뮤니티 템플릿
라이선스 사용자들끼리 스타일 템플릿(.md)을 공유하는 앱 내 게시판입니다.
마음에 드는 템플릿을 내 스타일 폴더에 바로 추가할 수 있습니다.
접근 방법
앱 [스타일] 탭 하단 [🌐 커뮤니티 템플릿] 버튼 클릭
주요 기능
📥 템플릿 가져오기
게시물 클릭 → 미리보기(MD 원문 + 네이버 스타일 렌더링) → [+ 내 스타일에 추가]
현재 활성 탭의 해당 카테고리 폴더에 자동 저장됩니다.
📤 템플릿 공유하기
[공유하기] 버튼 → 제목·카테고리·내용 입력 → 업로드.
라이선스 키가 자동 첨부되어 별도 로그인이 필요 없습니다.
🔍 카테고리 필터
도입부 / 요금제 / 개념_장점 / 마무리 등 카테고리별로 필터링해 원하는 템플릿을 찾을 수 있습니다.
👁 미리보기
MD 원문 확인 + 브라우저에서 네이버 스타일 렌더링 미리보기가 가능합니다.
도구
🔄 MD 변환기 (convert.html)
Claude나 Gemini로 만든 MD를 프로그램 규칙에 맞게 자동 교정하는 오프라인 도구입니다.
서버 불필요 — 브라우저에서 더블클릭으로 바로 실행됩니다.
📥 convert.html은 프로그램 zip 파일 안에 포함되어 있습니다.
기능 2가지
탭 1 — MD 변환기
• HTML 태그 → MD 자동 변환
• 이미지 배치 앞에 > 자동 추가
• 이중 헤딩 정규화
• 빈 불릿·연속 빈 줄 제거
• --- 구분선 자동 교정
탭 2 — 스타일 템플릿 만들기
섹션 설명을 입력하면 Gemini가 스타일 템플릿 .md 파일을 자동 생성합니다.

생성된 파일을 바로 다운로드해 styles/ 폴더에 넣으면 됩니다.
사용 방법
  1. 1
    convert.html 열기 naver-blog/convert.html을 브라우저로 열거나 더블클릭합니다.
  2. 2
    MD 변환기 탭 — AI 생성 MD 붙여넣기 ChatGPT나 Claude가 만든 MD를 왼쪽 텍스트박스에 붙여넣고 [변환]을 클릭합니다.
  3. 3
    교정된 MD 다운로드 오른쪽에 교정된 결과가 표시됩니다. [다운로드]로 .md 파일을 저장합니다.
