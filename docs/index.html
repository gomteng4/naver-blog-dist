프로그램 다운로드

zip 파일을 받아 압축을 풀면 바로 실행할 수 있습니다. 별도 설치 과정이 없습니다.

⬇  최신 버전 다운로드 약 110 MB
초기 설정 안내
  1. 1
    zip 다운로드 → 바탕화면에 압축 해제 위 버튼으로 zip 파일을 저장 후 바탕화면(또는 원하는 위치)에 압축 해제합니다.
    naver-blog/ 폴더가 생성됩니다.
  2. 2
    폴더 구조
    naver-blog/
      ├── naver-blog.exe    실행 파일
      ├── convert.html     MD 변환기 + 스타일 템플릿 만들기
      └── skills/styles/    기본 스타일 템플릿 (계정 추가 시 자동 복사)
  3. 3
    라이선스 키 입력 → 계정 추가 프로그램 실행 후 라이선스 키를 입력하고, [계정 추가] 버튼으로 네이버 아이디/비밀번호를 등록합니다.
    계정을 추가하면 콘텐츠/{아이디}/prompt/ 폴더와 기본 스타일 파일이 자동 생성됩니다.
  4. 4
    Gemini API 키 입력 (AI 생성 사용 시) 상단 [설정] 탭 → Gemini API 키를 입력합니다.
    Google AI Studio에서 무료로 발급받을 수 있습니다.
  5. 5
    Chrome이 자동 실행됩니다 포스팅 시작 시 Chrome이 자동으로 열리고 저장된 계정으로 네이버에 로그인한 뒤 포스팅합니다.
    첫 실행 시 Chrome 드라이버가 자동 설치됩니다.
블로그 ID 입력 (선택)

계정 추가 시 블로그 ID를 입력하면 정확한 포스팅이 가능합니다.
예: blog.naver.com/myid123 → 블로그 ID는 myid123
비워두면 네이버 아이디를 그대로 블로그 주소로 사용합니다.

AI 글 생성 안내

제목 목록과 작성 지침을 준비해두면, Gemini AI가 제목을 한 개씩 꺼내
블로그 글(MD)과 이미지 프롬프트를 동시에 자동 생성합니다.

준비 파일 — prompt/ 폴더 안에 작성
콘텐츠/{아이디}/
  └── prompt/
        ├── guide.md             브랜드 정보 + 금지사항 ← 필수 (계정 추가 시 자동 생성)
        ├── titles.txt            제목 목록 (한 줄에 하나) ← 필수
        ├── style_tabs.json     스타일 탭 설정 ← 프로그램 UI에서 자동 저장
        └── styles/                글 스타일 폴더 ← 계정 추가 시 자동 생성
                ├── 기본/                기본 스타일 탭
                │   ├── 도입부/          공감형.md, 질문형.md …
                │   ├── 요금제/          표형식.md, 리스트형식.md
                │   └── 마무리/          …
                └── 부동산/              추가 스타일 탭 (예시)
새 기능 — 스타일 탭 · 커뮤니티 (v1.4.25)
📁 스타일 탭 (프로필)
블로그 주제별로 독립된 스타일 프로필을 만들 수 있습니다.
탭 추가/이름 변경/삭제 — 스타일 탭 UI에서 관리.
📋 섹션 구성 UI
섹션 On/Off 및 ↑↓ 순서 변경.
작업 저장 시 현재 섹션 구성이 함께 저장됩니다.
✏️ 문체 선택
랜덤 / 친근한 / 전문적 / 간결한 중 선택.
작업셋트마다 다른 문체 저장 가능.
🌐 커뮤니티 템플릿
다른 사용자가 공유한 스타일 템플릿을 받아 내 스타일 폴더에 바로 추가할 수 있습니다.
titles.txt 예시
가경동 선불폰 개통 어떻게 하나요?
북가좌동 선불폰 KT망 vs LGU+망
명동 선불유심 약정 없이 쓰기 좋은 이유
신정동 선불폰 셀프개통 방법
… (한 줄에 제목 하나씩)
실행 흐름
  1. 1
    titles.txt에서 미사용 제목 1개 선택 순서대로 또는 무작위로 꺼냅니다. 모두 사용하면 알림이 표시됩니다.
  2. 2
    guide.md + 스타일 탭 템플릿 기반으로 Gemini가 글 작성 제목 + guide.md + 선택된 스타일 탭의 섹션 템플릿 → Gemini API → 블로그 본문 생성.
  3. 3
    이미지 프롬프트 자동 생성 블로그 본문의 이미지 배치([이미지 1]~[이미지 4])를 분석해 이미지 프롬프트를 자동 작성합니다.
  4. 4
    날짜 폴더 자동 생성 → 파일 저장 YYYYMMDD/ 폴더가 자동 생성되고 {제목}.md + image_prompts.txt가 저장됩니다.
    같은 날 여러 개 생성 시 YYYYMMDD_01/, YYYYMMDD_02/ 형식으로 번호가 붙습니다.
  5. 5
    이미지 생성 → 포스팅 실행 생성된 날짜 폴더를 선택하고 [포스팅 시작]을 누르면
    이미지 자동 생성 → 블로그 자동 업로드까지 한 번에 진행됩니다.
guide.md 준비 — 계정을 처음 추가하면 guide.md가 자동 생성됩니다.
아래 자료 다운로드에서 guide_만들기_프롬프트.md를 받아 ChatGPT·Claude·Gemini에 붙여넣고
내 서비스 정보를 채우면 완성된 guide.md가 자동으로 생성됩니다.
💡 포맷·구조 지시는 넣지 마세요 — 섹션 스타일(styles/)이 담당합니다.

✨ 스타일 탭 & 글 스타일 폴더 시스템

블로그 주제마다 스타일 탭(프로필)을 추가해 독립적인 섹션 구성을 관리할 수 있습니다.
각 탭의 styles/ 폴더에 .md 파일을 추가하면 AI 글 생성 시 랜덤 선택해 글 스타일을 다양하게 만듭니다.

guide.md vs styles/ 역할 분리
📄 guide.md — "무엇을"
  • 브랜드명, URL, 서비스명
  • 요금제 정보, 가격, 혜택
  • 타겟 독자, 금지사항
  • 브랜드 특징 (USP)
📁 styles/ — "어떻게"
  • 도입부 스타일 (공감/질문/상황)
  • 요금제 표시 형식 (표/리스트)
  • 개통 순서 구성 방식
  • 전체 구조, 문체, 섹션 포맷
⚠️ guide.md에 포맷 지시를 넣지 마세요.
"표로 정리해줘", "Q&A 형식으로 써줘" 같은 내용은 styles/ 폴더에 .md로 추가하세요.
guide.md와 styles/가 충돌하면 글 품질이 저하됩니다.
스타일 파일 관리 방법
  1. 앱 [스타일] 탭 → 스타일 프로필(탭) 추가/전환
  2. 섹션 목록에서 📂 폴더 버튼으로 탐색기 열기
  3. 폴더 안의 .md 파일을 수정하거나 새 .md 파일 추가
  4. [🌐 커뮤니티 템플릿] 버튼으로 다른 사용자 템플릿을 직접 가져올 수도 있습니다
