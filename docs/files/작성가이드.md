# 블로그 자동 포스팅 MD 파일 작성 가이드

이 프로그램은 MD 파일을 읽어 네이버 블로그에 자동으로 글을 올립니다.
아래 규칙을 지켜야 프로그램이 정상적으로 동작합니다.

---

## 1. 파일 인코딩

- **UTF-8 (BOM 없음)** 으로 저장
- BOM 있어도 자동으로 처리되지만, 없는 쪽이 안전함
- Windows 메모장 저장 시 "UTF-8" 선택 (UTF-8 BOM 아님)

---

## 2. 파일 위치 및 이름

```
콘텐츠/
  └── {아이디폴더}/
        └── {날짜폴더}/
              ├── {날짜-제목-슬러그}.md   ← 글 내용
              ├── image_1.png            ← 이미지 파일
              ├── image_2.png
              └── ...
```

- 날짜폴더 형식: `YYYY-MM-DD(순번)` 예) `2026-04-08(1)`
- MD 파일은 폴더 안에 `.md` 확장자 파일 1개만 있으면 됨
- 파일명은 자유롭게 지어도 됨

---

## 3. 프론트매터 (필수)

파일 **맨 첫 줄**부터 시작해야 합니다.

```
---
title: "글 제목을 여기에 작성"
---
```

**규칙:**
- 첫 줄은 반드시 `---` (하이픈 3개, 앞뒤 공백 없음)
- `title:` 은 반드시 포함 — 없으면 해당 폴더 **자동 스킵**됨
- 제목은 큰따옴표로 감싸는 것 권장 (특수문자 포함 시 필수)
- 닫는 `---` 다음 줄은 빈 줄

---

## 4. 본문 맨 위 구조 (권장 예시)

아래 예시처럼 인용구(`>`)로 시작하는 경우가 많습니다.

```
> 도입부 핵심 문장

본문 내용
...
```

- `>` 로 시작하는 줄은 **인용구 블록 + 구분선**으로 삽입됨 — 반드시 제목을 반복할 필요는 없음
- H1(`# 제목`)은 **블로그 제목 입력창**에만 입력되고 본문에서는 제거됨 — 작성해도 되고 생략해도 됨

---

## 5. 소제목과 이미지 배치 (핵심 규칙)

소제목과 이미지는 항상 이 순서로 작성합니다.

```
## 소제목 텍스트

> [이미지 N]

본문 내용...
```

**프로그램 처리 흐름:**
1. `## 소제목` → **인용구 블록** 삽입
2. 인용구 → 자동으로 **구분선(수평선)** 으로 탈출
3. `> [이미지 N]` → 구분선 바로 다음에 **이미지** 업로드
4. 이후 본문 내용 입력

> 이미지는 반드시 소제목 바로 아래에 위치시킬 것. 다른 위치에 넣으면 이미지가 엉뚱한 곳에 들어갈 수 있음.

### `###` 소소제목 → 볼드 텍스트

```
### 소소제목 텍스트
```

- `###` 은 **볼드(굵게)** 처리된 텍스트로 삽입됨 (인용구 없음)

---

## 6. 볼드 (굵게)

```
문장 중에 **강조할 텍스트** 이렇게 씁니다.
```

- `**텍스트**` 형식으로 감싸면 해당 부분만 **볼드** 처리됨
- 한 줄에 여러 개 사용 가능

---

## 7. 인용구 (직접 사용)

```
> 인용할 내용을 여기에 씁니다
```

- `>` 로 시작하는 줄은 **인용구 블록**으로 삽입됨
- 인용구 다음에 자동으로 **구분선** 추가됨
- `##` 소제목과 동일한 처리 방식

---

## 8. 이미지

```
> [이미지 1]
> [이미지 2]
```

- `image_1.png`, `image_2.png` ... 파일을 순서대로 사용
- `grid.png` 파일이 있으면 자동 4분할 → `image_1~4.png` 저장
- 지원 확장자: `png`, `jpg`, `jpeg`, `webp`
- 이미지 파일이 없으면 해당 이미지만 **건너뜀** (글 작성은 계속됨)
- **위치**: 반드시 소제목(`##`) 바로 아래 첫 줄에 작성

---

## 9. 강조 소제목 (H5)

```
##### 강조 소제목 텍스트
```

- `#####` (샵 5개) → **19pt + 볼드 + 색상** 서식으로 삽입됨 (색상은 앱 설정에서 선택, 기본: 파랑)
- 섹션 내 핵심 포인트를 강조할 때 사용
- 예: `##### 선불폰이 유리한 경우`, `##### 개통 전 준비사항`

---

## 10. 요금제명 (H4)

```
#### KT망 기본 요금제
```

- `####` (샵 4개) → **24pt + 볼드 + 색상 자동순환**으로 삽입됨
- 색상 순서: 빨강 → 파랑 → 진한핑크 → 주황 → 반복
- 요금제 이름처럼 크고 눈에 띄는 소제목이 필요할 때만 사용
- `####` 다음에 요금제 내용을 번호 리스트(`1. 2. 3.`)로 작성하는 게 권장 형식

```
#### KT망 기본 요금제
1. 데이터: 1GB
2. 월 요금: 8,800원
3. 통화/문자: 무제한
```

---

## 11. 번호 리스트 / 불릿 리스트

```
1. 첫 번째 항목
2. 두 번째 항목
3. 세 번째 항목
```

```
- 첫 번째 항목
- 두 번째 항목
- 세 번째 항목
```

- `1. 항목` → 네이버 블로그 **번호 리스트** 서식으로 삽입됨
- `- 항목` → 네이버 블로그 **불릿(동그라미) 리스트** 서식으로 삽입됨
- 연속된 항목은 자동으로 하나의 리스트로 묶임
- 리스트가 끝나면 자동으로 리스트 서식에서 탈출하여 이후 내용은 일반 텍스트

**주의:** `---` (구분선)은 `-` 항목과 다름. 하이픈 3개 단독 줄만 구분선으로 처리됨.

---

## 12. 구분선

```
---
```

- 하이픈 3개 이상 단독 줄 → **수평선** 삽입
- 소제목/인용구 다음 구분선은 **자동 삽입**되므로 직접 넣을 필요 없음
- 요금제와 요금제 사이 구분 등 추가 분리가 필요할 때 직접 사용

---

## 13. 표

```
| 항목 | 내용 | 비고 |
|------|------|------|
| 값1  | 값2  | 값3  |
```

- 마크다운 표 형식 그대로 사용
- 구분선 행(`|---|---|`)은 자동으로 제거됨

---

## 13. 목차 (선택)

```
## 목차
1. 소제목1
2. 소제목2
3. 소제목3
```

- `## 목차` 는 다른 `##` 소제목과 동일하게 **인용구 블록**으로 삽입됨 (+ 구분선 자동 삽입)
- 목차 항목(`1. 2. 3.`)은 **번호 리스트 서식**으로 자동 삽입됨
- 목차와 첫 번째 소제목 사이에 `---` 넣지 않음
- 앱에서 "목차 위치 링크" 설정 시, 목차 다음 첫 소제목 직전에 링크(위아래 구분선 포함) 자동 삽입됨

---

## 14. 전체 구조 예시

```
---
title: "선불폰 개통 방법, 이렇게 하면 됩니다"
---

> 선불폰 개통 방법, 이렇게 하면 됩니다

핵심 답변 1~2문장. 도입부 내용입니다.

## 목차
1. 선불폰이란
2. 요금제 안내
3. 개통 방법

## 선불폰이란

> [이미지 1]

소제목A 본문 내용입니다. **중요한 부분**은 볼드 처리합니다.

##### 선불폰이 유리한 경우

- 약정 없이 쓰고 싶은 분
- 신용 조회 없이 바로 개통이 필요한 분
- 세컨드폰이나 임시폰이 필요한 분

## 요금제 안내

> [이미지 2]

앤텔레콤 요금제를 확인하세요.

#### KT망 기본 요금제
1. 데이터: 1GB
2. 월 요금: 8,800원
3. 통화/문자: 무제한

---

#### LGU+망 매일5GB 요금제
1. 데이터: 매일 5GB (소진 후 1Mbps)
2. 월 요금: 22,000원
3. 통화/문자: 무제한

## 개통 방법

> [이미지 3]

세 단계로 완료됩니다.

1. 편의점에서 유심 구입
2. 이음모바일.COM 접속 후 인증
3. 요금제 선택 후 충전 완료

## 비교표

> [이미지 4]

| 구분 | 기본 | 5GB |
|------|------|-----|
| 데이터 | 1GB | 5GB |
| 요금 | 8,800원 | 22,000원 |
```

**실제 블로그에 올라가는 순서:**
1. 제목 입력창: `선불폰 개통 방법, 이렇게 하면 됩니다`
2. 본문: 인용구(제목) → 구분선 → 도입부
3. 목차 인용구 → 구분선 → **번호 리스트 서식** → (링크 설정 시 자동 삽입)
4. 소제목 인용구 → 구분선 → 이미지 → 본문
5. `#####` → **19pt + 볼드 + 앱 설정 색상** 강조 소제목
6. `- 항목` → **불릿 리스트** 서식
7. `#### 요금제명` → **24pt + 볼드 + 색상순환**
8. `1. 항목` → **번호 리스트** 서식

## 15. image_prompts.txt

이미지 생성 설정 파일입니다. MD 파일과 같은 폴더에 저장합니다.

### 작성 형식

```
[이미지 1]
프롬프트: Flat design infographic showing a SIM card...
텍스트: KT, 8,800원, 무약정

[이미지 2]
프롬프트: Flat design comparison cards...
텍스트: 기본1, 12,100원, 396, 39,600원

[썸네일]
프롬프트: Flat design thumbnail...
텍스트: 선불폰, 개통 가이드
```

**규칙:**
- `[이미지 N]` 번호는 MD의 `> [이미지 N]` 번호와 반드시 일치
- 프롬프트는 영어로 작성
- 텍스트 필드: 이미지에 삽입할 한글 키워드 (쉼표 구분)
- 썸네일이 필요 없으면 `[썸네일]` 섹션 생략

> 이미지 프롬프트 작성 상세 규칙 → `skills/이미지_프롬프트_지침.md` 참조

## 16. 주의사항 요약

| 항목 | 올바른 예 | 잘못된 예 |
|------|-----------|-----------|
| 인코딩 | UTF-8 | UTF-8 BOM |
| title | `title: "제목"` | title 없음 |
| 이미지 위치 | 소제목 바로 아래 | 본문 중간 임의 위치 |
| 이미지 태그 | `> [이미지 1]` | `[이미지1]`, `> [image 1]` |
| 이미지 파일명 | `image_1.png` | `img1.png`, `이미지1.png` |
| 소제목 | `## 소제목` | `# 소제목` (H1은 제목창으로 감) |
| 구분선 직접 사용 | 요금제 사이 등 추가 구분 시만 | 소제목/인용구 뒤 (자동 삽입됨) |
| 표 | 마크다운 표 형식 | HTML 표 |
| 요금제명 | `#### KT망 기본 요금제` | `## KT망 기본 요금제` |
| 요금제 내용 | `1. 데이터: 1GB` (번호 리스트) | 일반 텍스트 나열 |
| 강조 소제목 | `##### 개통 전 준비사항` | `#### 개통 전 준비사항` |
| 불릿 리스트 | `- 항목` (하이픈+공백) | `• 항목`, `* 항목` |
